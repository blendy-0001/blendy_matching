### **Phase 1: セキュリティ強化** ✅

- [x] **API キー・DB ID の環境変数化**
  - `config.py` で全て `os.getenv()` で読み込み
  - `.env` に設定値を記載
  - デフォルト値でローカル開発も可能

- [x] **CORS 設定**
  - FastAPI `CORSMiddleware` 導入
  - `ALLOWED_ORIGINS` 環境変数で制御
  - ローカル: `http://localhost:3000, http://localhost:8000`
  - 本番: `https://blendy-matching.onrender.com`

- [x] **認証・認可実装**
  - `verify_api_key()` 関数
  - 開発環境: スキップ（ENV=development）
  - 本番環境: X-API-Key ヘッダー必須（ENV=production）
  - 無効キー: 403 Forbidden

- [x] **ログレベル制御**
  - `LOG_LEVEL` 環境変数で制御
  - ローカル開発: DEBUG
  - 本番環境: INFO

- [x] **セキュリティヘッダー**
  - X-Content-Type-Options: nosniff
  - X-Frame-Options: DENY
  - X-XSS-Protection: 1; mode=block
  - Strict-Transport-Security: max-age=31536000

- [x] **エラー詳細隐蔽**
  - 詳細なトレーサーバックはサーバーログのみ
  - ユーザーには安全なエラーメッセージを返却

### **Phase 2: エラーハンドリング強化** ✅

- [x] **Notion API リトライロジック**
  - `tenacity` ライブラリ導入（8.2.3）

  - 最大 3 回まで自動リトライ
  - 対象関数: `get_all_members()`, `get_matched_pairs()`, `get_stats()`, `create_member()` など

- [x] **タイムアウト設定**
  - 全 Notion API 呼び出しに `timeout=10` 秒
  - タイムアウト時も自動リトライ

- [x] **例外処理の明確化**
  - `requests.exceptions.Timeout`
  - `requests.exceptions.RequestException`

  - 各々異なるログレベルで記録

- [x] **バックアップエラーハンドリング**

  - エラー時もデバッグログをバックアップ
