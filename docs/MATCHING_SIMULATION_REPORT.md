## 現在の課題 & 改善提案

### 課題 A: 高品質マッチが目標 (40%) に対し 26% に留まる

**原因分析:**
- 6メンバーのテストセットで、完全に相補的な組み合わせは限定的
- 特に「Designer」など単一スキル領域のメンバーが、複数領域メンバーとのマッチで低スコア化

**改善案:**
1. **テストデータの拡大**: 12-30メンバーでシミュレーション実施
   - 統計的に安定した分布が見える
   - より多くの相補性パターンが検出可能

2. **スコアリング重みの調整**:
   - 現在: Client(0-20) + Chain(0-20) + Market(0-20) + Expansion(0-20) + Target(0-20) = 100
   - 案: Chain と Market の重みを 25点 → 提案により多く

3. **潜在シナジー検知の活用** (Phase 0実装済み)
   - 45-60点の「低品質」ペアを LLM で再評価
   - 3つのヒューリスティック信号で LLM コストを削減
   - 実装済みなので有効活用

### 課題 B: 不適格 (<45点) ペアが 26% 存在

**特徴:**
- Member_A x Member_D: 35点 (Tech Dev x Designer)
  - 価値チェーン接続性は高い (20点) が、他の軸で補完性なし
  - **理由**: 両者とも営業課題を抱えており、相互補完性が低い

- Member_A x Member_C: 44点 (Tech Dev x DB Optimizer)
  - ポジション相補性はあるが、サービス対象業界がかぶらない
  - **理由**: 同じ「技術系」で、営業・マーケティング課題が共通

**改善案:**
- テストメンバーの多様性を増やす (営業×技術の組み合わせを増加)
- 実環境では、営業・マーケティング系メンバーが多いはずなので、自動的に改善される
