# Activities Database セットアップガイド

## 概要
フォーム送信で活動情報を Notion に保存するため、Activities テーブルを作成する必要があります。

## 手動セットアップ手順

### 1. Notion で新しいデータベースを作成

メンバーリスト（👥 メンバーリスト）と同じ場所に、以下の手順で新しいデータベースを作成してください：

1. Notion ワークスペースを開く
2. メンバーリストページ内で `+ Add a database` をクリック
3. データベース名を入力：**Activities**（またはアクティビティ）

### 2. データベースのプロパティを設定

以下のプロパティを追加してください：

| プロパティ名 | 型 | オプション |
|---|---|---|
| アクティビティ名 | Title（タイトル） | - |
| サービス内容 | Text（テキスト） | - |
| 対象業界 | Text（テキスト） | - |
| 対象企業規模 | Select（選択） | スタートアップ, 中小企業, 中堅企業, 大企業 |
| 強み | Multi-select（複数選択） | 技術・開発力, 営業・営業ネットワーク, 業界知見・実績, ブランド・認知度, コンテンツ・教材, その他 |
| 強み_詳細 | Text（テキスト） | - |
| 課題 | Multi-select（複数選択） | 営業・マーケティング力, 技術・開発力, 営業ネットワーク・顧客基盤, 資金・人材・リソース, 業界知見・実績, その他 |
| 課題_詳細 | Text（テキスト） | - |
| ポジション | Multi-select（複数選択） | 認知・ブランディング, 集客・マーケティング, リード獲得・見込み客育成, 営業・提案・クロージング, 制作・開発・導入, 運用・保守・継続支援, 教育・研修・人材育成, 経営・戦略・資金調達 |
| Member | Relation（関連） | メンバーリストを選択 |

### 3. データベース ID を取得

1. Notion で Activities データベースを開く
2. URL をコピー
3. URL から ID を抽出します：
   ```
   https://www.notion.so/[ID]?v=...
   ```

### 4. .env ファイルを更新

`.env` ファイルに以下を追加：

```
ACTIVITIES_DB_ID=[抽出したID（ハイフン付き）]
```

例：
```
ACTIVITIES_DB_ID=a1b2c3d4-e5f6-7890-abcd-ef1234567890
```

### 5. サーバーを再起動

```bash
# サーバーを停止
Ctrl+C

# サーバーを起動
python main.py
```

### 6. テスト

1. http://localhost:8000/register-multiactivity にアクセス
2. フォームを入力して送信
3. Notion で Activities データベースに新しいレコードが作成されることを確認

## トラブルシューティング

### "ACTIVITIES_DB_ID が見つかりません" エラーが表示される場合

- .env ファイルに `ACTIVITIES_DB_ID` が正しく設定されているか確認
- サーバーを再起動してから再度テスト

### フォーム送信後、Notion に記録が追加されない場合

- Notion インテグレーション（ブレンディーマッチング）が Activities データベースへのアクセス権を持っているか確認
- データベースの プロパティ名が正確に一致しているか確認（日本語の文字や空白に注意）
