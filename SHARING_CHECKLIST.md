# 申込フォーム共有前チェックリスト

外部ユーザーにフォームを送信する前に、必ずこのチェックリストを実施してください。

---

## ✅ 事前準備（初回のみ）

- [ ] Python がインストール済み
- [ ] ngrok がインストール済み
  ```bash
  ngrok --version
  ```
- [ ] ngrok アカウント認証完了
  ```bash
  ngrok config list
  ```
- [ ] 依存パッケージをインストール
  ```bash
  pip install -r requirements.txt
  ```
- [ ] `main.py` のカテゴリ情報は最新か確認

---

## 🚀 毎回の起動確認

### ステップ1: サーバー起動

**Windows:**
```
quick_share.cmd をダブルクリック
または
deploy.bat をダブルクリック
```

**Mac/Linux:**
```bash
python deploy.py
```

- [ ] エラーなく起動した

### ステップ2: 依存ツールの確認

- [ ] Python サーバーが起動 (ターミナルに "Uvicorn running" と表示)
- [ ] ngrok が起動 (ngrok ウィンドウが開いた)

### ステップ3: 接続確認

#### ローカル接続テスト
```
http://localhost:8001/register
```
にアクセスして、フォーム画面が表示されることを確認

- [ ] ロード完了
- [ ] フォーム入力フィールドが全て表示
- [ ] スクロール、ボタンが動作

#### ngrok URL 取得
1. ngrok ターミナルを見て、以下を確認:
   ```
   Forwarding  https://abc123.ngrok.io -> http://localhost:8001
   ```
2. または http://localhost:4040 で確認

- [ ] Forwarding URL が表示されている
- [ ] URL が `https://` で始まっている

### ステップ4: テスト送信（重要）

1. フォーム URL をブラウザで開く
   ```
   https://abc123.ngrok.io/register
   ```

2. テストデータを入力:
   - 名前: `テスト太郎`
   - 会社名: `テスト株式会社`
   - その他: 必須項目に何か入力

3. **「申込を送信」ボタンをクリック**

4. 以下いずれかが表示されることを確認:
   - [ ] 「✅ 申込が完了しました」メッセージ
   - [ ] エラーメッセージなし

5. **ダッシュボードで確認**
   ```
   https://abc123.ngrok.io/
   ```
   - [ ] 「総登録者数」が増えている
   - [ ] Notion データベースに記録が追加されている

### ステップ5: エラーハンドリング確認

1. **必須フィールド検証**
   - [ ] 「名前」を空で送信 → エラーメッセージ表示

2. **ネットワーク確認**
   - [ ] フォーム送信時にネットワークエラーが出ないか

3. **データベース接続**
   - [ ] Notion にデータが保存されているか

---

## 📤 外部ユーザーへの送信

### 送信前の最終確認

- [ ] テスト送信で完全に動作確認した
- [ ] URL が 24 時間有効か確認した
- [ ] 受信者に URL を送信する前に、スクリーンショット等で説明を準備した

### 送信内容テンプレート

```
【重要】協業マッチングシステム申込フォーム

以下のリンクから申込フォームにアクセスしてください：

📝 申込フォーム
https://abc123.ngrok.io/register

📊 ダッシュボード（マッチング結果確認）
https://abc123.ngrok.io/

【注意事項】
- このリンクは24時間のみ有効です
- 入力データは自動的にNotionに保存されます
- サポート: xxx@example.com
```

- [ ] URL が正しい
- [ ] 期限情報が明記されている
- [ ] 受信者が何をするか明確に書いている

## 🐛 トラブルシューティング

### もしエラーが出たら

#### "ポート 8001 が既に使用中" エラー

**Windows:**
```bash
netstat -aon | find ":8001"
taskkill /F /PID <PID>
```

**Mac/Linux:**
```bash
lsof -ti:8001 | xargs kill -9
```

その後、再度スクリプトを実行

- [ ] 解決した

#### "ngrok が見つからない" エラー

```bash
ngrok --version
```

で確認。インストールされていない場合:
https://ngrok.com/download からダウンロード

- [ ] ngrok インストール完了
- [ ] 認証完了: `ngrok config add-authtoken xxx`

#### フォーム送信時にエラー

1. ブラウザコンソール (F12) でエラーメッセージを確認
2. ターミナルの FastAPI ログを確認
3. Notion API キーが正しいか確認

**一般的なエラー:**
- 400 Bad Request → フォームデータ形式が不正（デバッグが必要）
- 500 Internal Server Error → Notion 接続エラー（API キー確認）
- 404 Not Found → URL が間違っている

---

## 📋 共有後の運用チェック

### 毎日の確認

- [ ] ダッシュボードで新規登録者数を確認
- [ ] エラーはないか確認
- [ ] Notion データベースに正しく記録されているか確認

### 24時間経過後

- [ ] URL が切れていることを確認
- [ ] 新たに共有する場合は、このチェックリストから開始

---

## ✨ 正常動作の目安

✅ こんな状態なら OK：

1. **フォームが表示される**
   - ページロードに 1-2 秒
   - 全フィールドが見える
   - 日本語が正常に表示される

2. **送信が完了する**
   - 送信ボタン → 3-5 秒で完了メッセージ
   - エラーメッセージがない
   - URL が長時間保持される（24時間）

3. **データが保存される**
   - Notion データベースに即座に記録
   - ダッシュボード統計が更新
   - 重複記録がない

4. **複数ユーザーがアクセス**
   - 同時に複数人がアクセスしてもエラーなし
   - 送信順序が保持される

---

**重要: このチェックリストを実施してから外部ユーザーに共有してください。**
