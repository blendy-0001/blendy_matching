## 1. 初期連絡メール（A さん向け）

**用途：** マッチング相手B さんを紹介するメール  
**送信タイミング：** マッチング結果確認後、当日〜翌日  
**媒体：** メール or LINE

---

**件名：** 💡 あなたの事業と相性の良いパートナーさんをご紹介です！

本文：

```
{A_NAME} さん、こんにちは！

ブレンドマッチのコーディネーターです。

このたび、{A_NAME} さんの事業と相性の良い、ぜひお話しいただきたいパートナーさんが見つかりました！

【ご紹介する方】
名前：{B_NAME} さん
会社：{B_COMPANY}
事業内容：{B_SERVICE}

【マッチングの理由】
{MATCHING_REASON}

つまり、{A_NAME} さんが「{A_STRENGTH}」という強みをお持ちの一方で、「{A_NEED}」という課題をお持ちとのこと。

一方 {B_NAME} さんは「{B_STRENGTH}」という強みをお持ちなので、ちょうど {A_NAME} さんの課題をカバーできる関係になります。

【提案する協業の形】
{COLLABORATION_TYPE}

逆に {A_NAME} さんが {B_NAME} さんの課題解決もできるので、win-win の関係を築けると思います。

まずは、30分程度のお時間をいただいて、お互い簡単に自己紹介していただけますか？
スケジュールが合えば、{COORDINATOR_NAME} が3者で電話 or 対面でご紹介させていただきたいのですが、いかがでしょうか？

お返事お待ちしています！

{COORDINATOR_NAME}
ブレンドマッチ
```

**記入例：**
- {A_NAME} → 「山田」
- {B_NAME} → 「鈴木」
- {B_COMPANY} → 「株式会社〇〇」
- {B_SERVICE} → 「BtoB向けSaaS営業支援ツール開発」
- {MATCHING_REASON} → 「山田さんのIT開発力とSaaS企業向けの営業ネットワークが、鈴木さんの新規顧客開拓課題をカバーできるため」
- {A_STRENGTH} → 「システム開発・技術力」
- {A_NEED} → 「営業・マーケティング力」
- {B_STRENGTH} → 「営業ネットワーク・顧客基盤」
- {COLLABORATION_TYPE} → 「B 市場参入型 - 鈴木さんのサービスを、山田さんのクライアント層に紹介」
- {COORDINATOR_NAME} → 「太郎」（あなたの名前）

## 7. NG集（避けるべき表現）

繋ぐ際に避けるべき表現をまとめました。信頼を損なわないよう、ご注意ください。

### ❌ これは避けましょう

**パターン1：一方的な紹介**
```
「{A_NAME}さんはすごい人なので、一緒にやってみてください」
「あなたの課題、{B_NAME}さんが解決できますよ」
```
→ 理由：相互性がない。まるで一方が他方を「助けてあげる」という感じになる。

**パターン2：スコアを強調しすぎ**
```
「スコア85点なので、相性が絶対にいいです」
「AIが計算したから間違いありません」
```
→ 理由：ビジネスは数値より人間関係。スコアを強調すると「AIまかせ」に見える。

**パターン3：プレッシャーをかける**
```
「同意していただきたいのですが」
「この機会を逃すと次はありません」
```
→ 理由：両者がまだ納得していない段階での圧力は、信頼を損なう。

**パターン4：曖昧な理由**
```
「相性がいいと思うので、会ってみてください」
「何か協業できるんじゃないかな」
```
→ 理由：具体性がないと相手は判断できない。「なぜこの人？」が明確でないと動きにくい。

**パターン5：ネガティブな課題描写**
```
「{A_NAME}さんは営業が下手なので」
「{B_NAME}さんには開発力がないから」
```
→ 理由：相手を傷つけ、マイナスイメージを植え付ける。「補完できる強み」として表現しよう。

---

### ✅ こう言いましょう

**パターン1：相互補完を明確に**
```
「{A_NAME}さんの{A_STRENGTH}と、{B_NAME}さんの{B_STRENGTH}が組み合わさると、
両者の課題をカバーできる関係が作れます」
```

**パターン2：スコアは背景情報として**
```
「当社のAIで分析した結果、相互に補完できるポイントが複数見つかりました」
```

**パターン3：ソフトなお誘い**
```
「まずは30分、お互い自己紹介してみませんか？
合わなければそれまでですし、話が盛り上がれば次のステップを考えましょう」
```

**パターン4：具体的な理由**
```
「{A_NAME}さんのクライアント層（{A_INDUSTRY}）に、
{B_NAME}さんのサービス（{B_SERVICE}）が非常にフィットしそうです。
理由は、そのクライアント層は今まさに『{PROBLEM}』という課題を抱えており、
{B_SERVICE}はそれを解決できるから」
```

**パターン5：ポジティブな表現**
```
「{A_NAME}さんが『{A_STRENGTH}』という強みをお持ちなので、
{B_NAME}さんの『{B_NEED}』という課題をカバーできます」
```

## 💡 ヒント

### 「繋ぐ」とは「信頼を介在させること」

ブレンドマッチのコーディネーターとしての役割は、単に「人を紹介する」ことではなく、
「両者の信頼を得た上で、相乗効果のあるパートナーシップを作る」ことです。

そのためには：

1. **相手を深く知ること** - 単なる属性情報ではなく、本当の課題・夢を理解する
2. **信頼を何度も確認すること** - 「合いそうですね」だけでなく、「なぜ合うのか」を対話で検証する
3. **長期的な関係を見ること** - 最初の契約ではなく、その後のWin-winな展開を想像する
4. **自分自身が「架け橋」になること** - AIのスコアではなく、あなたの言葉と判断が信頼の源
