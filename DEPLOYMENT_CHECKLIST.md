# 本番環境デプロイメントチェックリスト

協業マッチングシステムを本番環境に展開する際に確認すべき項目です。

## 1. セキュリティチェック ✅ CRITICAL

### 1.1 環境変数設定の確認
- [ ] `.env` ファイルがリポジトリに commit されていないことを確認
- [ ] `.gitignore` に `.env` が記載されていることを確認
  ```bash
  grep -n "\.env" .gitignore
  ```
- [ ] 本番環境の Render/デプロイプラットフォームに全ての環境変数が設定されている
  - [ ] `NOTION_API_KEY` (Notion インテグレーション API キー)
  - [ ] `CLAUDE_API_KEY` (Anthropic Claude API キー)
  - [ ] `MEMBERS_DB_ID` (Notion メンバーリスト DB ID)
  - [ ] `MATCHING_HISTORY_DB_ID` (Notion マッチング履歴 DB ID)
  - [ ] `MATCHING_RESULTS_DB_ID` (Notion マッチング結果 DB ID)
  - [ ] `UNMATCHED_MEMBERS_DB_ID` (Notion 未マッチメンバー DB ID)
  - [ ] `ALLOWED_ORIGINS` (CORS許可オリジン)
  - [ ] `API_KEY` (API認証キー)
  - [ ] `LOG_LEVEL` (ログレベル = `INFO` または `WARNING`)

### 1.2 API キー認証
- [ ] API キーが安全に保存されている（Render Secrets など）
- [ ] API キーが定期的に交換される予定がある
- [ ] 本番環境の API キーは開発環境と異なることを確認
- [ ] API キー交換手順が文書化されている

### 1.3 Notion インテグレーション権限
- [ ] Notion インテグレーションがすべての必要な DB にアクセス可能か確認
  ```
  必要な権限:
  - メンバーリスト DB: Read + Create/Update
  - マッチング履歴 DB: Read + Create
  - マッチング結果 DB: Read + Create
  - 未マッチメンバー DB: Read + Create (オプション)
  ```
- [ ] インテグレーションに不要な権限がないことを確認

### 1.4 CORS 設定
- [ ] `ALLOWED_ORIGINS` が本番ドメインに設定されている
- [ ] ローカル開発ドメイン (`localhost`) が本番設定に含まれていないことを確認
- [ ] 例: `ALLOWED_ORIGINS=https://blendy-matching.onrender.com`

### 1.5 セキュリティヘッダー
- [ ] Security headers ミドルウェアが有効であることを確認
  - [ ] X-Content-Type-Options: nosniff
  - [ ] X-Frame-Options: DENY
  - [ ] X-XSS-Protection: 1; mode=block
  - [ ] Strict-Transport-Security: max-age=31536000

### 1.6 ログレベル設定
- [ ] 本番環境で `LOG_LEVEL=INFO` または `LOG_LEVEL=WARNING` に設定
- [ ] DEBUG ログが本番環境で有効になっていないことを確認

## 2. エラーハンドリングと信頼性チェック ✅

## 実行確認スクリプト

本番環境準備完了を確認するワンライナー：

```bash
#!/bin/bash
echo "=== 本番環境デプロイメント準備確認 ==="
echo "1. .env がバージョン管理外か？"
git check-ignore .env && echo "✓ OK" || echo "✗ NG: .env が commit 対象になっています"

echo "2. requirements.txt が最新か？"
grep "tenacity" requirements.txt > /dev/null && echo "✓ OK" || echo "✗ NG: tenacity が未記載"

echo "3. テストはパスしているか？"
pytest tests/ -q && echo "✓ OK" || echo "✗ NG: テストが失敗しています"

echo "4. ログレベルが設定されているか？"
grep "LOG_LEVEL" .env.example > /dev/null && echo "✓ OK" || echo "✗ NG: .env.example に LOG_LEVEL がありません"

echo "=== チェック完了 ==="
```

使用方法：
```bash
bash deploy_check.sh
```
